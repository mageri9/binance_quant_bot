# MarketMind

> **MarketMind** — это платформа для исследований алгоритмической торговли и машинного обучения, которая автоматически собирает рыночные данные, обучает модели, отслеживает качество их работы, калибрует риск-менеджмент и проводит реалистичную paper trading торговлю.

Это не бот с сигналами.

Это инженерная ML-система, построенная вокруг воспроизводимости экспериментов, автоматизации и непрерывного развития моделей.

---

# Почему появился MarketMind

Большинство любительских торговых проектов выглядят примерно одинаково:

```
Скачать котировки

↓

Обучить модель

↓

Сохранить model.pkl

↓

Надеяться на прибыль
```

Такой подход плохо масштабируется.

Невозможно понять:

* на каких данных обучалась модель;
* какие признаки использовались;
* какая версия датасета дала лучший результат;
* почему качество ухудшилось спустя месяц;
* когда модель перестала соответствовать рынку.

MarketMind решает именно эти проблемы.

Здесь модель является лишь одной частью системы.

# Возможности

## Сбор рыночных данных

* загрузка исторических свечей Binance;
* автоматическое обновление данных;
* безопасный Upsert без дубликатов;
* асинхронная работа через CCXT;
* хранение истории в SQLite.

---

## Feature Engineering

Во время подготовки датасета автоматически рассчитываются признаки:

* RSI
* MACD
* Bollinger Bands
* ATR
* ADX
* Volatility
* Volume Ratio

Архитектура позволяет без изменений основного кода добавлять новые признаки.

---

## Генерация целевых переменных

Поддерживаются два режима обучения.

### Бинарная классификация

```
Рост

Падение
```

### Трёхклассовая классификация

```
LONG

HOLD

SHORT
```

Горизонт прогнозирования и пороги полностью настраиваются.

# Построение датасетов

Каждый датасет автоматически получает собственный паспорт.

В нем сохраняются:

* торговая пара;
* таймфрейм;
* версия;
* список признаков;
* параметры генерации целей;
* диапазон данных;
* Git-коммит проекта;
* дата создания.

Благодаря этому любой эксперимент можно воспроизвести даже спустя несколько месяцев.

---

# Обучение моделей

Каждое обучение сопровождается сохранением:

* параметров модели;
* используемых признаков;
* версии датасета;
* Git SHA;
* итоговых метрик.

В проекте отсутствует концепция "непонятного model.pkl".

Любая модель имеет собственную историю происхождения.

---

# Walk-Forward Validation

Для проверки качества используется хронологическое разделение данных.

```
Обучение
████████████████

Тест
                ████

Будущее
                    ████
```

Такой подход значительно ближе к реальным условиям торговли, чем случайное разделение выборки.

---

# Автоматическая калибровка риск-менеджмента

Stop Loss и Take Profit не задаются вручную.

После обучения система самостоятельно подбирает наиболее эффективные параметры, используя перебор комбинаций на исторических данных.

Оптимизация выполняется по:

* Sharpe Ratio;
* Sortino Ratio;
* Profit Factor;
* Expectancy;
* Win Rate;
* общей доходности.

Это позволяет отказаться от случайных значений риска.

---

# Paper Trading

Встроенный движок виртуальной торговли поддерживает:

* Long-позиции;
* Short-позиции;
* расчёт размера позиции;
* учёт свободного капитала;
* стоимость открытых позиций;
* расчёт прибыли и убытка;
* историю всех сделок.

Таким образом можно оценивать модель в условиях, максимально приближенных к реальному рынку.

---

# Логирование прогнозов

Каждый прогноз модели автоматически сохраняется.

Фиксируются:

* версия модели;
* вероятности всех классов;
* итоговый прогноз;
* цена актива;
* время создания прогноза.

Это позволяет анализировать качество моделей задним числом и отслеживать изменение их поведения.

---

# Контроль экспериментов

Все результаты обучения сохраняются в базе данных.

Для каждого эксперимента фиксируются:

* параметры обучения;
* итоговые метрики;
* версия датасета;
* Git SHA.

История развития моделей никогда не теряется.

---

# Контроль деградации модели

Рынок постоянно меняется.

MarketMind умеет обнаруживать изменение распределений признаков (Concept Drift), что позволяет своевременно переобучать модели и избегать незаметного ухудшения качества прогнозов.

# Философия проекта

Модель машинного обучения сама по себе не представляет ценности.

Ценность появляется тогда, когда вокруг неё построена система.

В MarketMind ключевыми принципами являются:

* воспроизводимость экспериментов;
* автоматизация всего жизненного цикла модели;
* контроль качества данных;
* измеримость каждого этапа;
* возможность безопасного переобучения;
* расширяемая архитектура.

Проект создаётся как исследовательская платформа, которую можно постепенно развивать от paper trading до полноценной алгоритмической торговли.
